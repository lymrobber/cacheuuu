16 ## Levels
17 
18 IoTgo Arduino Library has levels following:
19 
20  - IoTgo Device Level: Switch/Light/THSensor [source: Dev_*]
21  - Http Proxy Level: IoTgo [source: IoTgo*]
22  - Ethernet Level: ESP2560(WIFI) [source: Eth_*]
23  - Hardware Level: WBoard/UNO/MEGA

25 # Configuration and Connection
26 
27 ## Configure Your Board
28 
29 When you use with UNO board, uncomment the follow line in `Eth_ESP8266.h`.
30 
31  #define UNO
32 
33 When you use with MEGA board or WBoard, uncomment the follow line in `Eth_ESP8266.h`.
34 
35  #define MEGA
36 
37 ## Connection
38 
39 ### UNO
40 
41 When you use it with UNO board, the connection should be like these:
42 
43  ESP8266_TX->D0
44 
45  ESP8266_RX->D1
46 
47  ESP8266_CHPD->3.3V
48 
49  ESP8266_VCC->3.3V
50 
51  ESP8266_GND->GND
52 
53  //The baud rate of software serial can't be higher that 19200, so we use software serial as a debug port
54  FTDI_RX->D3
55 
56  FTDI_TX->D2
57 
58 ### MEGA
59 
60 When you use it with MEGA board, the connection should be like these:
61 
62  ESP8266_TX->RX1(D19)
63 
64  ESP8266_RX->TX1(D18)
65 
66  ESP8266_CH_PD->3.3V
67 
68  ESP8266_VCC->3.3V
69 
70  ESP8266_GND->GND
71 
72 When you want to output the debug information, please use DebugSerial. For example,
73 
74  DebugSerial.println("hello");
75 
76 ### WBoard
77 
78 Need no connections.

80 # Attention
81 
82 **Note1**: The size of message from ESP8266 is too big for arduino sometimes,
83 so the library can't receive the whole buffer because the size of the hardware
84 serial buffer which is defined in HardwareSerial.h is too small.
85 
86 Open the file from `\arduino\hardware\arduino\avr\cores\arduino\HardwareSerial.h`.
87 See the follow line in the HardwareSerial.h file.
88 
89  #define SERIAL_BUFFER_SIZE 64
90 
91 The default size of the buffer is 64. Change it into a bigger number, like 256
92 or more.
93 
94 The SRAM size of mega is bigger than UNO's, so it is better to use MEGA board to
95 communicate with ESP8266.
